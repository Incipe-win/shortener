## Project Overview

Smart-Shortener is an intelligent URL shortener gateway built with Go (go-zero) backend and React frontend. It features JWT authentication, Redis caching, Kafka async event processing, LLM-based URL safety analysis, and OpenTelemetry tracing.

## Architecture

### High-Level Flow

```
User → incipe.top:2053 (HTTPS)
  → Nginx (TLS termination + rate limiting)
    → /api/* → Go backend (:8888)
    → /:short_url → Go backend (302 redirect)
    → /* → React frontend SPA (static files)
```

### Backend Architecture (go-zero)

The backend follows go-zero's layered architecture:

```
internal/
  config/        - Config struct (MySQL DSN, Redis, Kafka, LLM, JWT, CORS)
  svc/           - ServiceContext: wires all dependencies together
  handler/       - HTTP handlers, thin layer that delegates to logic
  logic/         - Business logic (convert, show, preview, auth, links, stats)
  middleware/    - JWT cookie auth (required + optional), CORS, Redis rate limiting
  consumer/      - Kafka consumers: AI analysis, click events, safety alerts
  types/         - Shared Go types
  ctxdata/       - Context key constants
```

**ServiceContext** (`internal/svc/serviceContext.go`) is the central dependency injection point. It holds:
- `ShortUrlModel`, `UserModel`, `SequenceModel` — MySQL model interfaces (generated by goctl + manual extensions)
- `Filter` — Redis Bloom Filter for short URL existence checks
- `LLMClient` — LLM client for AI URL analysis (nil if LLM disabled)
- `KafkaProducer` — Async event producer (nil if Kafka disabled)
- `BlackListDomains` — Domain blacklist for safety checks

### Key Architectural Patterns

1. **Async Event Processing**: URL clicks are written to Kafka, then asynchronously consumed and batch-updated to MySQL via `consumer/clickConsumer.go`. This decouples hot-path redirects from heavy DB writes.

2. **Bloom Filter**: All existing short URLs are loaded into a Redis Bloom Filter at startup to quickly reject non-existent URLs before hitting MySQL.

3. **JWT Cookie Auth**: Tokens are stored in HTTP-only cookies. Two middleware variants exist:
   - `JWTCookieMiddleware` — requires valid token (returns 401 if missing/invalid)
   - `OptionalJWTCookieMiddleware` — token is optional; injects empty user context if absent

4. **Kafka Consumers**: Three consumer groups run alongside the HTTP server, managed by `service.ServiceGroup` for unified lifecycle:
   - `ai-analysis` — LLM-powered URL content analysis (summary, keywords, risk rating)
   - `click-events` — Async click count aggregation
   - `safety-alerts` — Safety alert notifications

### Frontend Architecture (React)

```
web/src/
  pages/        - Route-level components
  components/   - Reusable UI components
  stores/       - Zustand state stores
  lib/          - Utilities and API client
  types/        - TypeScript type definitions
  router.tsx    - TanStack Router configuration
  App.tsx       - Root component
```

State management uses **Zustand**, data fetching uses **TanStack React Query**, routing uses **TanStack Router**. UI styled with **TailwindCSS v4** and **Framer Motion** for animations. Charts use **Recharts**.

### Data Layer

- **MySQL 8.4**: Primary datastore. Model code in `model/` — partially generated by `goctl`, partially hand-written extensions.
- **Redis 7**: Caching (go-zero cache pattern), Bloom Filter, rate limiting.
- **Kafka**: Async event bus for click tracking, AI analysis, and safety alerts.

### SQL Migration Files

Executed automatically on first container startup via `docker-entrypoint-initdb.d/`:
1. `short_url_map.sql` — Base short URL mapping table
2. `sequence.sql` — Sequence generator table
3. `alter_short_url_map.sql` — Table schema extensions
4. `alter_click_count.sql` — Click count fields
5. `alter_user.sql` — User table + user ID fields

### API Routes

**Public** (with CORS + rate limiting):
- `GET /health` — Health check
- `POST /convert` — Create short URL (unregistered users limited to 3)
- `GET /convert/remaining` — Remaining creation count for unregistered users
- `GET /preview/:short_url` — URL preview with AI summary + risk rating
- `GET /:short_url` — Short URL redirect (302)
- `POST /auth/login`, `POST /auth/register`, `POST /auth/logout`

**Authenticated** (with CORS + JWT):
- `GET /auth/me` — Current user info
- `GET /links` — User's link list (paginated + searchable)
- `GET /stats` — Dashboard statistics

**Monitoring**:
- `GET /metrics` — Prometheus metrics endpoint
- `GET /debug/pprof/*` — Go pprof profiling endpoints (internal port :8888 only, not exposed via Nginx)

## Important Notes

- The `shortener.api` file defines API contracts for go-zero's `goctl` code generator. After modifying it, regenerate handler stubs with `goctl api go -api shortener.api`.
- Model files ending in `_gen.go` are auto-generated by goctl; do not edit them manually. Custom model methods go in the corresponding non-`_gen` file.
- Config is loaded from `etc/shortener-api.yaml` with environment variable substitution (`conf.UseEnv()`).
- The `BaseString` config field controls the base62 character set for short URL encoding.
- Frontend dev proxy is configured in `web/vite.config.ts` to route `/api/*` and `/auth/*` to the Go backend.
- pprof endpoints are registered in `shortener.go` but NOT exposed through Nginx — only accessible on the internal `:8888` port for security.
- Benchmarking infrastructure uses `docker-compose.bench.yaml` overlay to add Prometheus + Grafana alongside the base stack. Grafana dashboards are auto-provisioned from `scripts/bench/grafana/provisioning/`.
- Request duration histograms are recorded in `convertHandler`, `showHandler`, and `previewHandler` via `metrics.RequestDuration`. Queryable as `shortener_request_duration_seconds_bucket` in Prometheus.
