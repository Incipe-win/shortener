## Project Overview

Smart-Shortener is an intelligent URL shortener gateway built with Go (go-zero) backend and React frontend. It features JWT authentication, Redis caching, Kafka async event processing, LLM-based URL safety analysis, and OpenTelemetry tracing.

## Development Commands

### Backend (Go)

```bash
# Run locally (requires MySQL + Redis + Kafka running first)
go run shortener.go

# Build binary
CGO_ENABLED=0 go build -ldflags='-s -w' -o bin/server .

# Run tests
go test ./...

# Run tests for a specific package
go test ./internal/logic/...
go test ./pkg/base62/...
```

### Frontend (React)

```bash
cd web
pnpm install
pnpm dev        # Dev server with hot reload + API proxy to :8888
pnpm build      # Production build
pnpm lint       # Biome + ESLint
```

### Docker / Deployment

```bash
make docker-build   # Build all Docker images
make up             # Start all services (nginx, backend, frontend, mysql, redis, kafka, jaeger)
make down           # Stop all services
make logs           # View service logs
make restart        # Restart all services
make backup         # MySQL backup
make clean          # Tear down volumes + prune Docker system
```

### Local Development Setup

```bash
# 1. Start infrastructure only
docker compose up -d mysql redis kafka jaeger

# 2. Run backend locally
go run shortener.go

# 3. Run frontend locally
cd web && pnpm dev
```

Backend listens on `0.0.0.0:8888`, frontend dev server on `0.0.0.0:3000` (with API proxy).

## Architecture

### High-Level Flow

```
User ‚Üí incipe.top:2053 (HTTPS)
  ‚Üí Nginx (TLS termination + rate limiting)
    ‚Üí /api/* ‚Üí Go backend (:8888)
    ‚Üí /:short_url ‚Üí Go backend (302 redirect)
    ‚Üí /* ‚Üí React frontend SPA (static files)
```

### Backend Architecture (go-zero)

The backend follows go-zero's layered architecture:

```
internal/
  config/        - Config struct (MySQL DSN, Redis, Kafka, LLM, JWT, CORS)
  svc/           - ServiceContext: wires all dependencies together
  handler/       - HTTP handlers, thin layer that delegates to logic
  logic/         - Business logic (convert, show, preview, auth, links, stats)
  middleware/    - JWT cookie auth (required + optional), CORS, Redis rate limiting
  consumer/      - Kafka consumers: AI analysis, click events, safety alerts
  types/         - Shared Go types
  ctxdata/       - Context key constants
```

**ServiceContext** (`internal/svc/serviceContext.go`) is the central dependency injection point. It holds:
- `ShortUrlModel`, `UserModel`, `SequenceModel` ‚Äî MySQL model interfaces (generated by goctl + manual extensions)
- `Filter` ‚Äî Redis Bloom Filter for short URL existence checks
- `LLMClient` ‚Äî LLM client for AI URL analysis (nil if LLM disabled)
- `KafkaProducer` ‚Äî Async event producer (nil if Kafka disabled)
- `BlackListDomains` ‚Äî Domain blacklist for safety checks

### Key Architectural Patterns

1. **Async Event Processing**: URL clicks are written to Kafka, then asynchronously consumed and batch-updated to MySQL via `consumer/clickConsumer.go`. This decouples hot-path redirects from heavy DB writes.

2. **Bloom Filter**: All existing short URLs are loaded into a Redis Bloom Filter at startup to quickly reject non-existent URLs before hitting MySQL.

3. **JWT Cookie Auth**: Tokens are stored in HTTP-only cookies. Two middleware variants exist:
   - `JWTCookieMiddleware` ‚Äî requires valid token (returns 401 if missing/invalid)
   - `OptionalJWTCookieMiddleware` ‚Äî token is optional; injects empty user context if absent

4. **Kafka Consumers**: Three consumer groups run alongside the HTTP server, managed by `service.ServiceGroup` for unified lifecycle:
   - `ai-analysis` ‚Äî LLM-powered URL content analysis (summary, keywords, risk rating)
   - `click-events` ‚Äî Async click count aggregation
   - `safety-alerts` ‚Äî Safety alert notifications

### Frontend Architecture (React)

```
web/src/
  pages/        - Route-level components
  components/   - Reusable UI components
  stores/       - Zustand state stores
  lib/          - Utilities and API client
  types/        - TypeScript type definitions
  router.tsx    - TanStack Router configuration
  App.tsx       - Root component
```

State management uses **Zustand**, data fetching uses **TanStack React Query**, routing uses **TanStack Router**. UI styled with **TailwindCSS v4** and **Framer Motion** for animations. Charts use **Recharts**.

### Data Layer

- **MySQL 8.4**: Primary datastore. Model code in `model/` ‚Äî partially generated by `goctl`, partially hand-written extensions.
- **Redis 7**: Caching (go-zero cache pattern), Bloom Filter, rate limiting.
- **Kafka**: Async event bus for click tracking, AI analysis, and safety alerts.

### SQL Migration Files

Executed automatically on first container startup via `docker-entrypoint-initdb.d/`:
1. `short_url_map.sql` ‚Äî Base short URL mapping table
2. `sequence.sql` ‚Äî Sequence generator table
3. `alter_short_url_map.sql` ‚Äî Table schema extensions
4. `alter_click_count.sql` ‚Äî Click count fields
5. `alter_user.sql` ‚Äî User table + user ID fields

### API Routes

**Public** (with CORS + rate limiting):
- `GET /health` ‚Äî Health check
- `POST /convert` ‚Äî Create short URL (unregistered users limited to 3)
- `GET /convert/remaining` ‚Äî Remaining creation count for unregistered users
- `GET /preview/:short_url` ‚Äî URL preview with AI summary + risk rating
- `GET /:short_url` ‚Äî Short URL redirect (302)
- `POST /auth/login`, `POST /auth/register`, `POST /auth/logout`

**Authenticated** (with CORS + JWT):
- `GET /auth/me` ‚Äî Current user info
- `GET /links` ‚Äî User's link list (paginated + searchable)
- `GET /stats` ‚Äî Dashboard statistics

**Monitoring**:
- `GET /metrics` ‚Äî Prometheus metrics endpoint

## Important Notes

- The `shortener.api` file defines API contracts for go-zero's `goctl` code generator. After modifying it, regenerate handler stubs with `goctl api go -api shortener.api`.
- Model files ending in `_gen.go` are auto-generated by goctl; do not edit them manually. Custom model methods go in the corresponding non-`_gen` file.
- Config is loaded from `etc/shortener-api.yaml` with environment variable substitution (`conf.UseEnv()`).
- The `BaseString` config field controls the base62 character set for short URL encoding.
- Frontend dev proxy is configured in `web/vite.config.ts` to route `/api/*` and `/auth/*` to the Go backend.
